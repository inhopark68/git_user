# OCR_SQLite 개발 단계 정리 (기초 → 중급 → 고급)

본 문서는 OCR_SQLite 프로젝트의 전체 개발 과정을 단계별로 정리한
문서입니다.\
각 단계는 **구조 안정화 → 영속화 → 확장성(EAV)** 순으로 설계됩니다.

------------------------------------------------------------------------

# 1️⃣ 기초 단계 (Starter Level)

## 🎯 목표

-   FastAPI 서버 기동
-   React(Vite) 프론트 연결
-   CORS 문제 없이 API 호출
-   파일 업로드 및 기본 저장
-   문서 목록 조회

------------------------------------------------------------------------

## 📁 Backend 최소 디렉터리 구조

    backend/
      requirements.txt
      app/
        __init__.py
        main.py
        config.py
        storage/

### 필수 기능

-   `/api/v1/health`
-   `/api/v1/ocr` (파일 업로드)
-   `/api/v1/documents`
-   `/storage` 정적 파일 서빙
-   In-memory 저장 구조 (초기 단계)

------------------------------------------------------------------------

## 📁 Frontend 최소 디렉터리 구조

    frontend/
      package.json
      vite.config.ts
      index.html
      tsconfig.json
      src/
        main.tsx
        App.tsx
        api.ts

### 필수 기능

-   health 체크
-   이미지 업로드
-   문서 목록 조회
-   Vite proxy 설정 (`/api`, `/storage`)

------------------------------------------------------------------------

## 🚀 실행 방식

    launcher/
      launch.bat
      stop.bat
      launch.ps1
      stop.ps1

원클릭 실행으로 backend + frontend 동시 실행

------------------------------------------------------------------------

# 2️⃣ 중급 단계 (Persistence Level)

## 🎯 목표

-   SQLite 영속 저장
-   서버 재시작 후 데이터 유지
-   문서 상세 조회 API
-   표준화된 에러 응답

------------------------------------------------------------------------

## 추가 구성

### Backend 확장

    backend/app/
      db.py
      models.py
      sqlite_db.py

### 주요 기능

-   SQLite 연결
-   documents 테이블 생성
-   `/api/v1/documents/{id}`
-   DB 기반 CRUD 구조
-   로그 구조 정리

------------------------------------------------------------------------

## 기술 변화

  항목     기초        중급
  -------- ----------- -----------
  저장     메모리      SQLite
  구조     단일 파일   모듈 분리
  확장성   낮음        보통

------------------------------------------------------------------------

# 3️⃣ 고급 단계 (EAV + OCR 확장 Level)

## 🎯 목표

-   문서 유형별 동적 필드 관리
-   템플릿 기반 구조 (EAV)
-   bbox 매핑
-   Export 기능
-   실무형 구조 완성

------------------------------------------------------------------------

## 📁 Backend 고급 구조

    backend/app/
      eav/
        templates.py
        documents.py
        models.py
        services.py

------------------------------------------------------------------------

## 주요 API

    /api/eav/templates
    /api/eav/documents/{id}/fields
    /api/eav/documents/{id}/extract
    /api/eav/documents/{id}/map-field-bbox

------------------------------------------------------------------------

## 기능 확장

-   OCR block 기반 좌표 매핑
-   템플릿 자동 추출
-   CSV / JSON Export
-   최소 인증(API Key)
-   Smoke Test

------------------------------------------------------------------------

# 📊 단계별 비교 요약

  단계   저장           구조        확장성   대상
  ------ -------------- ----------- -------- -----------------
  기초   메모리         단일 구조   낮음     학습/프로토타입
  중급   SQLite         모듈화      중간     실사용 준비
  고급   SQLite + EAV   계층화      높음     실무/배포용

------------------------------------------------------------------------

# 🏁 권장 개발 순서

1.  Starter 안정화
2.  SQLite 영속화 적용
3.  EAV 모듈 분리
4.  OCR bbox 매핑
5.  Export + 인증
6.  Android/Web 확장

------------------------------------------------------------------------

# 🔚 결론

OCR_SQLite는 다음과 같은 성장 구조를 가집니다:

Starter → Persistence → Structured Extraction → Enterprise Extension

단계를 명확히 나누어 개발하면 구조가 깨지지 않으며,\
유지보수 및 확장이 쉬운 아키텍처를 유지할 수 있습니다.
